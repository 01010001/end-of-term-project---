  Bu iki hatanın sebebi çok netti ve her ikisini de kernel (C) kodunda tek seferde düzelttim:

  1. Tarihin "1970" Gözükme Sorunu (Timestamp Bug)
  Unix sistemlerde zaman 0 ise 1 Ocak 1970 (Epoch) kabul edilir. Inode ilk yaratıldığında (vcfs_new_inode fonksiyonunda) version_id = 0 set ediliyordu
  ancak version_timestamp değerini o anki Linux saati ile doldurmayı atlamıştık. Ayrıca file.c içindeki yazma (write) fonksiyonu bitiminde de (write_end)
  saati güncellemiyordu. Artık v0 (ilk versiyon) dahil tüm versiyonlar modifiye edildiği anın tam saat bilgisini Kernel'den alacak.

  2. v0'ın "Boş (Empty)" Gözükme Sorunu (Snapshot Bug)
  Terminalde echo "test" > dosya.txt yaptığında Linux çekirdeği (VFS) arkada şu adımları izler:
   1. İçi boş bir Inode (dosya) yarat. (Boyut: 0, Versiyon: 0)
   2. Dosyayı O_WRONLY (yazma) formatında aç.
   3. "test" kelimesini içine yaz.

  Hatırlarsan biz Copy-on-Write mantığını "dosya yazma formatıyla (O_WRONLY) açıldığında snapshot al" şeklinde kurgulamıştık. Bu durumda modülümüz daha
  3. adıma (dosyaya test yazılması) geçilmeden 2. adımda 0 byte'lık (içi boş) v0 versiyonunu snapshot olarak gizli listeye ekliyordu! "test" yazısı ise
  bir sonraki v1 versiyonuna kaydediliyordu. 

  Bunu önlemek için file.c dosyasına tam olarak şu satırları ekledim:

   1     /* Do not create an empty snapshot for a brand new file */
   2     if (ci->version_id == 0 && inode->i_size == 0)
   3         return 0;

  düzenlemek istersen (ör: echo "test2" > dosya.txt) o zaman v0 snapshot'ı alınıp, v1 üzerinden devam edilecek.
